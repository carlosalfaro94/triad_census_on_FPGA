Detailed Description

This file contains the definitions of the functions needed to create and manage an edge of a graph. within project, we consider only directed graphs; edges must have a direction.
In our implementation, edge information is saved twice, since each of the nodes keeps its adjacency list (see node.h).
Each edge structure is therefore part of the node it belongs to, and contains the id of the other neigbhbor, and the direction.
The direction is coded using the 2 less significant bits of the uint32_t used for storing the edge.
'01' means that the edge goes out from the node it belongs, '10' means edge goes in, and '11' conveys bidirectionality.
