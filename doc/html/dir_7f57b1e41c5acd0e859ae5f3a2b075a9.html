file  edge.h [code]
 This file contains the definitions of the functions needed to create and manage an edge of a graph. within project, we consider only directed graphs; edges must have a direction.
In our implementation, edge information is saved twice, since each of the nodes keeps its adjacency list (see node.h).
Each edge structure is therefore part of the node it belongs to, and contains the id of the other neigbhbor, and the direction.
The direction is coded using the 2 less significant bits of the uint32_t used for storing the edge.
'01' means that the edge goes out from the node it belongs, '10' means edge goes in, and '11' conveys bidirectionality.
 
file  graph.h [code]
 This file contains the definitions of the functions needed to create and manage a graph.
In our implementation, the graph is read from a file which, in each of its lines, contains an edge of the graph in this format:
src_node_id tgt_node_id
In the structure, nodes can be ordered by ascending order of their ids.

 This file contains the definitions of the functions

In our implementation, the graph is basically a list of nodes
which are identified uniquely with and unsigned integer. Each
node keeps track of its adjacency list, bi means of an array of
edges. Since each edge connects two nodes, edge information will
be duplicated.
